Para resolver problemas complejos usando código y diseño, nos apoyamos en 4 bases fundamentales[cite: 46, 73, 83]:

### 1. Descomposición 
**¿Qué es?:** Agarrar un problema gigante y desarmarlo en partes chiquitas y fáciles de manejar
**En Diseño:** Si hay que visualizar algo como la "Brecha Salarial", no se programa todo de un viaje


### 2. Reconocimiento de Patrones 
**¿Qué es?:** Buscar tendencias o repeticiones en el problema para automatizarlas
**En Diseño:** Si necesitas mostrar a 100 personas, no vas a dibujar 100 veces el mismo elemento

### 3. Abstracción
**¿Qué es?:** Filtrar lo que no sirve y quedarse solo con lo esencial para crear una representación simbólica de la realidad
**En el Código:** Se traduce en **Variables** y en la función **`map()`** Por ejemplo, la posición real del mouse (`mouseX`) se "abstrae" para transformarla en un valor de opacidad o de miedo

### 4. Algoritmos 
* ]**¿Qué es?:** Armar una serie de reglas ordenadas (un plan de acción) para que el sistema resuelva el problema[cite: 83, 84].
**En el Código:** Se traduce en **Diagramas de Flujo** y en **Condicionales **
